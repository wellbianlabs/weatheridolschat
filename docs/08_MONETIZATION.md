# 08 · Monetization

## 1. 수익 모델 3대 축

```
┌─────────────────────────────────────────────────────────────┐
│ 1. Freemium 구독            (Direct, 마진 가장 높음)         │
│ 2. Affiliate (Nasmedia)     (Indirect, 트래픽 기반)          │
│ 3. Web3 토큰 / 리워드 생태계 (Engagement loop, retention)    │
└─────────────────────────────────────────────────────────────┘
```

## 2. Freemium

### 2.1 플랜 비교

| 항목 | Free | Premium |
| :-- | :-- | :-- |
| 대화 모델 | Gemini Flash (빠름, 가벼움) | Claude 3.5 Sonnet (깊이) |
| 일 메시지 한도 | 30 | 무제한 (안전선 1,000) |
| 이미지 생성 | 3/일 | 50/일 |
| 장기 기억 | ✕ (최근 20개 메시지만) | ✓ (memory_summary 사용) |
| 광고/상품 카드 | 표시 | 표시 (선택 토글로 제거 가능) |
| 캐릭터 전용 코스튬 | 기본 | 시즌 코스튬 해금 |
| 화상통화/노래/안무 (Phase 2) | ✕ | ✓ |
| 가격 (KR) | ₩0 | ₩9,900/월, ₩99,000/년 (2개월 무료) |
| 가격 (글로벌) | $0 | $7.99/월, $79.9/년 |

### 2.2 게이팅 트리거 (페이월 노출 지점)

| 트리거 | UX |
| :-- | :-- |
| Free에서 일 메시지 30회 도달 | "오늘 써니와의 대화가 끝났어요. 더 이야기하려면 ✨" |
| Free에서 이미지 4번째 요청 | 인라인 카드 + "프리미엄으로 무제한" |
| 캐릭터가 메모리 기반 답변 시도 | 답변 자리에 잠금 카드 + "내가 너를 기억하게 하려면" |
| 화상통화/노래 (Phase 2) | 메뉴 항목에 PRO 뱃지 |

### 2.3 결제 라우팅

| 플랫폼 | 결제 수단 | 비고 |
| :-- | :-- | :-- |
| iOS app | StoreKit2 IAP (Apple) | 30%(첫해)/15% 수수료 |
| Android app | Google Billing | 동일 |
| Web | Stripe (글로벌) / 토스페이먼츠 (KR) | 수수료 ~3% |
| 외부 결제 우회 | Apple/Google 정책 위반 — 시도하지 않음 |

웹 결제 → `subscriptions.tier=premium` 갱신 → 모바일에서도 동일 효과.

### 2.4 환불 / 해지

- 자동 갱신 7일 전 푸시·이메일 안내.
- 정기결제 해지: 즉시 처리, 잔여기간 유지.
- 한국 전자상거래법: 결제 7일 내 100% 환불(미사용 시).

### 2.5 트라이얼 / 프로모션

- 신규: 첫 결제 7일 무료체험.
- 시즌: 캐릭터 데뷔일 기념 첫달 50%.
- 친구 초대: 양쪽 모두 +1주 프리미엄.

## 3. Affiliate (Nasmedia 연동)

### 3.1 사업 구조

```
[유저] ──대화──▶ [캐릭터 응답]
                       │ "오늘 이거 어때?"
                       ▼
                  [ProductCard]
                   │ impression / click
                   ▼
         [Nasmedia Tracking URL]
                   │ CPC / CPS postback
                   ▼
              [정산 / 보고]
```

### 3.2 노출 규칙 (Native Commerce)

- 캐릭터 추천 도메인에 매칭되는 캠페인만 노출:
  - sunny → outdoor/fitness/beauty
  - rain → cafe/healing/book
  - cloudy → art/photo_spot/stationery
  - thunder → delivery/game/streetwear
- **빈도 제한:**
  - 채팅 세션 1개당 최대 1 카드 / 10 메시지
  - 직전 카드 클릭 안 했으면 30분간 동일 캐릭터 추천 보류
- **컨텍스트 매칭:** 사용자 발화에 [추천/뭐먹/어디갈] 의도가 감지되었을 때 우선.
- **시간 매칭:** 식사 시간(11~13, 17~20)에 F&B 가중.
- **날씨 매칭:** 비 → 우산·실내 카페, 더운 날 → 음료·아이스크림.

### 3.3 디자인 원칙

- "광고" 라벨 명시 (KR 전상법) — 카드 하단 마이크로 텍스트.
- 캐릭터 카피로 래핑("써니의 추천 ✨").
- 외부 링크는 새 창(웹) / 인앱 브라우저(앱) — Universal Link 우선.

### 3.4 트래킹 데이터

- `recommendation_events` 테이블에 impression/click/conversion 기록.
- Nasmedia가 postback 호출 시 conversion 매칭(`message_id` + `transaction_id`).
- 일 1회 정산 잡: 어제 click 수, CTR, eCPM, conversion 수 집계.

### 3.5 사용자 옵션

- 설정 → "상품 추천 받기" 토글.
- 끄면 ProductCard 노출 전면 중단(프리미엄 가입자는 기본 OFF로 시작).

## 4. Web3 리워드 (Wellbian 토큰)

### 4.1 단계적 도입

| Phase | 형태 |
| :-- | :-- |
| MVP | 오프체인 가상 토큰 (DB `token_ledger`). 출금 불가, 인앱 사용만. |
| Phase 2 | 인앱 사용처(코스튬 해금, 이미지 추가권) 확대 |
| Phase 3 | 실 토큰 컨트랙트 연동 (KAIA/Klaytn 또는 Base) + KYC + 출금 옵션 |

### 4.2 적립 방식

| 액션 | 보상 |
| :-- | :-- |
| 신규 가입 | +100 |
| 출석(일 1회) | +5 |
| 기상 제보 퀘스트 (위치 + 사진) | +30 |
| 친구 초대 성공 | +100 |
| 광고 카드 클릭 | +5 (일 최대 +20) |
| 광고 카드 구매 전환 | +상품가의 0.5% (상한 1,000) |
| 캐릭터 별 누적 대화 시간 (10분 단위) | +1 |

### 4.3 사용처 (Sink)

| 항목 | 비용 |
| :-- | :-- |
| 이미지 추가 1회 (free 사용자) | 50 |
| 시즌 코스튬 임시 24h 체험 | 80 |
| 프리미엄 1주 체험권 | 700 |
| 캐릭터 보이스 메모 (Phase 2) | 30 |

### 4.4 규제 & 위험

- KR: 가상자산이용자보호법 적용 가능성 → 출금 가능 시점부터 KYC + 사업자 신고.
- 광고 보상은 도박·페이드 리뷰 우려 → "구매 전환 보상"은 자체 검토 + 약관 고지.
- 미성년자 토큰 거래: 출금 단계에서 만 19세 미만 차단.

## 5. 가격·과금 데이터 모델 보강

- `subscriptions` 테이블 + Stripe/Toss/IAP webhook 통합.
- `usage_counters` (option: Redis): `messages_today`, `images_today` 캐시.
- 매일 00:00 KST `pg_cron`으로 카운터 리셋.

## 7. KPI / 재무 시뮬레이션 (개념)

| 지표 | MVP 가정 |
| :-- | :-- |
| MAU | 5,000 (3개월차) |
| 무료→유료 전환율 | 3% |
| ARPPU | ₩9,900 |
| 월 구독 수익 | 5,000 × 3% × 9,900 = ₩1,485,000 |
| 광고 eCPM | ₩2,500 |
| 광고 노출/유저/월 | 80 |
| 월 광고 수익 | 5,000 × 80 × 2,500 / 1,000 = ₩1,000,000 |
| 합계 | ≈ ₩2.5M/월 |

→ Phase 2(영상/음악) 도입 시 ARPPU 약 1.6x 가정.
